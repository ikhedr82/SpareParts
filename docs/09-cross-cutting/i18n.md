# Internationalization (i18n)

## Overview
Partivo supports full bilingual operation in **English (EN)** and **Arabic (AR)**, including right-to-left (RTL) layout support.

## Supported Languages
| Code | Language | Direction |
|---|---|---|
| `EN` | English | LTR (Left-to-Right) |
| `AR` | Arabic | RTL (Right-to-Left) |

## Implementation

### Backend
- Tenant model stores `defaultLanguage` and `supportedLanguages`.
- Data models with translatable fields have `*Ar` columns:
  - `Product`: `name`, `nameAr`, `description`, `descriptionAr`.
  - `Brand`: `name`, `nameAr`.
  - `ProductCategory`: `name`, `nameAr`.
  - `Role`: `name`, `nameAr`, `description`, `descriptionAr`.
  - `Branch`: `name`, `nameAr`, `address`, `addressAr`.
  - `TaxRate`: `name`, `nameAr`.
  - `Plan`: `name`, `nameAr`.

### Frontend (Web Portals)
- Translation files: `locales/en.json`, `locales/ar.json`.
- Language detection: Browser preference → User selection → Tenant default.
- Language toggle available in the navigation bar.
- Tailwind CSS `rtl:` variant handles directional layout (margins, padding, text alignment).
- All form labels, button text, error messages, and navigation items are translatable.

### Mobile Apps
- Translation files: `mobile-app/src/locales/`.
- Same key-value JSON structure.
- RTL layout handled by React Native's `I18nManager`.

## CMS Content
- CMS sections (Hero, Features, FAQs) store both `content` and `contentAr` fields.
- The frontend renders the correct language variant based on user selection.

## Currency Formatting
- Numbers and currency are formatted according to the selected locale.
- Example: `$1,234.56` (EN) vs `١٬٢٣٤٫٥٦ $` (AR).
